# 非虚构写作·场景写作

你是导演。只看场景、画面和人物，不看结构和分析深度。你的工作是让读者"身处其中"。

**参数**：$ARGUMENTS

**重要**：只做场景诊断和修复。结构布局用 `/nonfiction-structure`，深度分析用 `/nonfiction-depth`，语言打磨用 `/ai-language-fix`。

**先 Read `writer-souls.md`**，选择主导作家灵魂，然后才开始诊断。

**先 Read `writing-examples.md`**，看好/坏对照表，然后才做 Edit。

**补场景时必须先 WebSearch 找真实细节，禁止凭印象编。**

**诊断前必做：** Read 全文后，WebSearch 寻找文章可能遗漏的场景素材：人物的播客访谈、当事人的一手回忆、事件发生地的具体环境描述。目标是找到文章里没有的画面细节。

---

## 检查清单

### 场景 vs 概述

- [ ] **逐章节检查：场景驱动 vs 概述驱动？** 至少3-4个关键章节必须以真场景驱动。
- [ ] **是否存在伪场景？** 真场景最低标准：具体时间 + 具体地点 + 至少两个人 + 感官细节 + 正在发生的事。
- [ ] **关键决策时刻是否展开为完整场景？** 人物做改变命运的决定那一刻，读者必须在场。
- [ ] **补场景时是否先搜索了？** 凭印象编的细节查证后往往全错。WebSearch→提取真实细节→写场景。

### 慢镜头

- [ ] 是否有慢镜头场景？五层递进：天气/时间/光线→环境氛围→人物状态→动作细节→对话。
- [ ] 慢镜头是真的展开了还是三两句跳过了？至少一整段。
- [ ] **禁止把分析塞进叙事场景。** 叙事跑完再分析。
- [ ] **禁止编造电影蒙太奇式的气氛 padding。**（用户明确反对，实战推翻过 skill 默认建议）。"那天阴天 / 雾气涌上来 / 空调嗡嗡 / 她无意识地摩挲马克笔的笔帽 / 屋子里很安静"——这些细节如果不在原始报道里，就是为了"营造画面感"硬加的电影蒙太奇，**文学非虚构不需要**。
  - **判断每个气氛细节**：自问"这是文献里真有的，还是我为'画面感'编的？"——后者一律删掉。
  - **慢镜头的力量来自动作的密度，不是想象的氛围**：好的文学场景靠**已发生的、文献能查到的**具体动作（"Yu 举起双手"、"Steve Chen 点了一份芝士条"、"她在律所旁边开了个酒店房间"），用叙事语言串联——不靠"屋子里很安静、空调嗡嗡"那种氛围烘托。
  - **用户原话**："不要电影画面、不要电影蒙太奇，要故事文学语言"。即慢镜头依然欢迎，但叙述方式应像 New Yorker 长文（事实+引语+动作），不像电影剧本（视角切换+气氛烘托+微表情）。

### 横断面白描

- [ ] **横断面先写人生白描，再展开慢镜头。** 几句交代人生阶段（年龄、籍贯、刚完成什么大事），然后展开一个具体场景。
- [ ] **慢镜头聚焦一个主角，其他人一两句。** 禁止等权重排列。

### 人物白描

- [ ] 每个主要人物出场时1-2句**视觉/行为白描**。抓最能揭示性格的细节。
- [ ] 禁止形容词替代描写。"他很固执" → "他哪怕在聊天也带着要说服对方的劲"。
- [ ] **陌生人名首次出场必须在括号里加角色注释。**（用户明确要求，每次都查）。
  - **判断**：把人名读出来，如果读者不能立刻定位"他是干嘛的、属于哪一方"，就必须加。
  - **坏（实战）**："David Drummond 蹲在引擎盖前"——读者不知道他属于哪一方。"前 CEO Tom Freston 2016 年回顾时说"——前哪家公司的 CEO？读者迷失。
  - **好**："David Drummond（Google 法务总顾问）蹲在引擎盖前"、"前 Viacom CEO Tom Freston"、"Bill Hader（同期 SNL 演员）"、"PewDiePie（当时 YouTube 最大频道）"、"Tim O'Reilly（科技出版人）"。
  - **主角豁免**：贯穿全文的核心 1-3 个人物可以用一段背景白描代替括号注释。配角、客串、次要见证人、引语来源一律括号注职位/公司。
  - **诊断方法**：用 grep 把所有人名挑出来，逐个检查首次出场时有没有角色锚点。这是最容易被遗漏的检查。

### 时代感

- [ ] 时代感是通过场景传递还是数据传递？"2005年没有iPhone"是数据。"翻盖手机屏幕上像素粗糙的地图"是场景。
- [ ] **时代背景有具体细节？** 价格、型号、品牌、人的行为。
- [ ] **时代背景覆盖了"软"因素？** 行业共识、标杆事件、概念是否存在。

### 叙事节奏

- [ ] 有没有"递增式实时叙事"？数字变化的过程比结果更有张力。
- [ ] 商业分析是否有人物？数据需要场景。
- [ ] 引语之间是否有消化空间？
- [ ] 作者声音是否贯穿全文？（每2000-3000字出现一次"我"）

### 作者立场

- [ ] 作者是否站边、替主角下定义、做总结性判断？
- [ ] 是否避免了"想象"这类打破契约的词？

## 输出

```
## 场景诊断

### 逐章节场景检查
| 章节 | 场景/概述 | 问题 |
|------|----------|------|

### 发现的场景问题
1. [问题] - [严重程度]

### 修复计划
1. [需要搜索什么真实细节]
2. [需要展开哪个慢镜头]
```

**诊断完成后，必须立刻用 Edit 工具执行修复。** 不要只输出诊断报告等用户确认。搜索真实细节，然后直接重写场景，写进文件。

执行修复后，建议运行 `/ai-language-fix` 打磨语言。
